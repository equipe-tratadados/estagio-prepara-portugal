# Biblioteca - [CÓDIGO DA SUBTAREFA, ex: DU002.1]

> **Instruções:**
> 1. Uma linha por fonte investigada.
> 2. **Não edite o `biblioteca.md` diretamente.** Este ficheiro é consolidado pela líder.
> 3. Antes de adicionar uma linha, verifique se a fonte já existe no `biblioteca.md`. Se existir, não duplique — avise a líder para atualizar a entrada.
> 4. Mensagem de commit sugerida: `docs(DU002.1): adiciona fonte [Nome da fonte]`
>
> **Notas por coluna:**
> 1. **Fonte ID:** use um ID provisório (ex: DU002.1-A, DU002.1-B). O ID final (F0XX) é atribuído pela líder na consolidação.
> 2. **Nome:** use o nome oficial da fonte. Evite siglas sozinhas (ex: `INE – Estatísticas do Comércio Internacional`, não apenas `INE`).
> 3. **Link direto:** URL que leva diretamente à página de descrição ou download dos dados. Evite links genéricos (ex: página inicial do INE).
> 4. **Período coberto:** indique o intervalo de datas (ex: `2010–2024` ou `desde 2000`).
> 5. **Frequência de atualização:** use `anual`, `mensal`, `trimestral`, `diária`, `sem atualização prevista` ou `Não informada`.
> 6. **Múltiplos itens** (Dados disponibilizados, Formato de export, Limitações): separe com `<br>` para não partir a tabela (ex: `Excel<br>CSV`).
> 7. **Responsável:** nome completo + link GitHub (ex: `[Nome Sobrenome](https://github.com/username)`).
> 8. Se uma coluna não se aplicar, escreva `Não se aplica`. Nunca deixe em branco.
